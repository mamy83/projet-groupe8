# Cahier des Charges - SolarGuard

## 1. Présentation du Projet

### 1.1 Contexte
En Guinée, l'accès limité à l'électricité pousse de nombreux foyers à adopter des kits solaires. Cependant, la gestion inefficace de l'énergie stockée réduit la durée de vie des batteries et limite l'autonomie énergétique.

### 1.2 Objectif
SolarGuard est une application web qui optimise l'utilisation de l'énergie solaire en analysant la consommation et en conseillant l'utilisateur sur le meilleur moment pour utiliser ses appareils électriques.

### 1.3 Technologies utilisées
- **Framework** : Django (Python)
- **Base de données** : SQLite (développement) / PostgreSQL (production)
- **Frontend** : Django Templates, CSS, JavaScript
- **Déploiement** : PythonAnywhere ou Heroku

---

## 2. Fonctionnalités Principales

### 2.1 Gestion du profil utilisateur
- Inscription et connexion
- Configuration du système solaire (capacité batterie, panneaux, onduleur)
- Liste des appareils électriques avec leur consommation (Watts)

### 2.2 Suivi de l'énergie
- Saisie manuelle du niveau de batterie (%)
- Affichage de l'autonomie restante estimée
- Historique de consommation sur 7 jours

### 2.3 Recommandations intelligentes
- Conseils sur les appareils à utiliser ou éviter selon le niveau de batterie
- Alertes quand la batterie atteint un seuil critique (ex: 20%)
- Suggestions pour prolonger la durée de vie des batteries

### 2.4 Statistiques simples
- Graphique de consommation hebdomadaire
- Temps d'utilisation total par appareil
- Économies réalisées grâce aux recommandations

---

## 3. Architecture Simplifiée

### 3.1 Base de données
Quatre tables principales :
- **Utilisateurs** : profil et authentification
- **Systèmes solaires** : caractéristiques du kit installé
- **Appareils** : liste des équipements électriques
- **Relevés** : historique des niveaux de batterie

### 3.2 Algorithme de recommandation
Logique en Python qui :
1. Calcule l'énergie disponible (Wh)
2. Évalue la consommation potentielle par appareil
3. Classe les appareils en : **Recommandé** / **Acceptable** / **À éviter**
4. Génère des conseils personnalisés

---

## 4. Parcours Utilisateur

### 4.1 Premier usage
1. Inscription avec email et mot de passe
2. Configuration du système (ex: batterie 200Ah, panneau 300W)
3. Ajout des appareils (ex: TV 60W, Ventilateur 50W, Ampoules 20W)

### 4.2 Utilisation quotidienne
1. L'utilisateur entre le niveau actuel de batterie (ex: 75%)
2. Le système affiche :
   - Autonomie restante (ex: "12 heures avec usage modéré")
   - Appareils recommandés (ampoules, téléphone)
   - Appareils à éviter (fer à repasser, climatiseur)
3. L'utilisateur consulte ses statistiques de consommation

---

## 6. Contraintes et Limitations

### 6.1 Données
- Saisie manuelle des niveaux de batterie (pas de capteurs connectés pour la V1)
- Base de données pré-remplie avec appareils courants

### 6.2 Accessibilité
- Optimisé pour connexions Internet lentes
- Interface disponible en français uniquement (V1)
- Compatible avec téléphones de base (pas d'application mobile native)

### 6.3 Partenariats
- Phase pilote avec 1-2 installateurs locaux
- Feedback utilisateurs sur 50 foyers tests

- Application fonctionnelle accessible en ligne
